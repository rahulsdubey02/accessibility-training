1.2.3 Audio Description or Media Alternative (Prerecorded)

An alternative for time-based media or an audio description of the prerecorded video content is provided for synchronised media — except when the media is a text alternative and clearly labelled as such.

Intent

This criterion ensures that blind users can access visual information in prerecorded synchronised media (video with audio). Unlike SC 1.2.5 (Level AA), 1.2.3 gives authors a choice of two approaches:

  • Audio description — a narration track added during natural pauses in dialogue that describes important visual information (actions, settings, scene changes, on-screen text)
  • Full media alternative — a text document that describes everything in the video in sequential order, combining both transcript and visual description

Either option satisfies this Level A requirement. SC 1.2.5 (AA) requires the audio description approach specifically — the text alternative is no longer sufficient at that level.

Key Distinctions

Audio Description vs. Transcript
A transcript only captures speech. An audio description narrates visual elements not covered by speech — what is shown on screen, who is in frame, on-screen text, meaningful gestures. The media alternative required here must cover both.
Applies Only When Audio Doesn't Already Describe the Visual
If the existing audio track fully explains what is happening visually — as in a lecture where the speaker describes every chart — no audio description is needed. The visual information is already conveyed via the audio.
Media Alternative Exemption
If the video is itself a media alternative for existing text content on the page and is clearly labelled as such, this criterion does not apply. The text provides the equivalent and the video is supplemental.

Examples

Example 1 of 4

Tutorial Video with Visual Steps — No Description

A software tutorial shows the presenter clicking through menus and configuring settings visually. The voiceover says "click here, then select this option" without naming what is shown. A blind user following along has no way to identify what is on screen.

Failure: F67 Technique: G69
✗ Poor Practice
Video element — no description provided
0:32 / 2:10
HTMLFAIL
<video controls>
  <source src="tutorial.mp4"
          type="video/mp4">
  <!-- F67: no audio description
       and no media alternative -->
</video>
🔊 A blind user hears"Now click here and select this option from the menu" — no information about what menu, what is highlighted, or what the UI shows

Problem (F67): The audio track references visual elements without naming them. Blind users cannot follow the tutorial because the key information exists only on screen.

✓ Good Practice
Media alternative — full text description

Video: Configuring Notification Settings

0:00 — The Settings screen opens, showing five menu items in a vertical list: Profile, Notifications, Privacy, Appearance, and Security.

0:08 — The cursor moves to "Notifications" and clicks it. The Notifications panel slides in from the right.

0:14 — A toggle labelled "Email Alerts" is currently off. The narrator says: "Switch this on." The toggle is clicked and turns blue.

HTMLPASS
<!-- G69: full media alternative -->
<video controls
       aria-describedby="tutorial-alt">
  <source src="tutorial.mp4"
          type="video/mp4">
</video>

<section id="tutorial-alt"
         aria-label="Full video description">
  <h3>Video: Configuring Notification Settings</h3>
  <p><strong>0:00</strong> — The Settings screen opens,
  showing five menu items: Profile,
  Notifications, Privacy, Appearance,
  Security.</p>
  <p><strong>0:08</strong> — Cursor moves to
  "Notifications" and clicks. The
  Notifications panel slides in.</p>
  <!-- continues for full video -->
</section>
🔊 A blind user reads"0:00 — The Settings screen opens, showing five menu items: Profile, Notifications, Privacy…" — every visual action is described in sequence

Fix (G69): A full media alternative document is provided on the same page, describing both the audio content and the visual actions in time-stamped sequence. This satisfies 1.2.3 at Level A.

Audio Description Track via <track> Element

A documentary film contains extended sequences of visual-only storytelling — archival footage with no narration. An audio description track narrates what is shown during natural pauses or silent passages.

Fix (G78 + H96): A <track kind="descriptions"> element references a WebVTT file containing narration of visual content timed to appear during natural pauses in the audio.

Remediation Notes

⚠️

Browser support for kind="descriptions" is incomplete

Most browsers do not natively play description tracks as audio. Screen reader users can read the cue text, but the audio description experience requires a player that explicitly supports it (e.g., AblePlayer). Always provide a text media alternative as a guaranteed fallback.

⏸️

Descriptions must fit in existing pauses

Standard audio description inserts narration during natural pauses without pausing the video. If there is insufficient time in the pauses to describe complex visual content, extended audio description (SC 1.2.7) pauses the video to allow longer narration.

Example 3 of 4

Video Where Audio Already Describes the Visual Content

An educational lecture recording features a presenter who explicitly describes every slide they show: "This bar chart shows three bars — the tallest on the left represents Q1 at 42%…" No additional audio description is needed because the visual information is already conveyed through speech.

Exception case Technique: G78
✗ Poor Narration — Requires Description
Transcript excerptFAIL
Speaker audio (paraphrased):

"As you can see from this chart,
Q1 performed significantly better
than the other quarters."

<!-- Problem: "as you can see" assumes
     sighted viewing. The data values,
     bars, and comparisons are NOT
     conveyed in the audio. -->
🔊A blind user hears"As you can see from this chart, Q1 performed significantly better…" — no actual data, no values, the chart content is completely lost

Problem: Phrases like "as you can see" and "significantly better" do not describe the visual. An audio description is required to convey the actual chart data.

✓ Self-describing Audio — No Extra Description Needed
Transcript excerptPASS
Speaker audio (paraphrased):

"This bar chart shows four bars.
Q1 is the tallest at 42%. Q2 drops
to 31%. Q3 recovers slightly at 35%.
Q4 reaches 38%."

<!-- No audio description needed:
     all chart data is stated aloud.
     1.2.3 is satisfied by the
     existing audio track. -->
🔊A blind user hears"This bar chart shows four bars. Q1 is the tallest at 42%. Q2 drops to 31%…" — all visual data is conveyed directly through speech

Exception: When the narrator explicitly describes all visual content in the audio track, no separate audio description is required. Design video scripts to be self-describing.

Example 4 of 4

Linked Media Alternative — Separate Page

A marketing video showcasing a product is embedded on a page. Rather than providing a long description on the same page, the team links to a separate page containing a full media alternative document combining the transcript and visual description.

Technique: G58 Technique: G69
✗ Poor Practice
HTMLFAIL
<video controls>
  <source src="product-video.mp4">
</video>

<!-- A link to the transcript exists
     but only provides the spoken words,
     not the visual content -->
<a href="transcript.html">
  Read the transcript
</a>

<!-- Missing: description of what
     is shown visually on screen -->

Problem: Providing only a speech transcript is not a media alternative — it omits the visual information. The link text "transcript" does not indicate it covers visual content, and the linked document only contains dialogue.

✓ Good Practice
HTMLPASS
<figure>
  <video controls>
    <source src="product-video.mp4">
  </video>
  <figcaption>
    <!-- G58: adjacent link to alt -->
    <a href="product-media-alt.html">
      Full media alternative for
      this video (transcript with
      visual description)
    </a>
  </figcaption>
</figure>

<!-- product-media-alt.html contains
     both the dialogue AND descriptions
     of all visual content, in sequence -->
🔊A blind user reads the link"Full media alternative for this video (transcript with visual description)" — purpose is clear; linked page provides complete sequential description of both audio and video

Fix (G58 + G69): The link is adjacent to the video and its label makes clear it is a full media alternative — not just a transcript. The linked document covers visual content in addition to dialogue.
