1.2.5 Audio Description (Prerecorded)

Audio description must be provided for all prerecorded video content in synchronised media. Unlike SC 1.2.3 (Level A), a text media alternative is no longer sufficient — an actual audio description track is required at this Level AA threshold.

Intent

Audio description (AD) provides a spoken narration of visual information that is not already conveyed through the existing audio track. It is inserted during natural pauses in dialogue to describe actions, scene changes, on-screen text, and other meaningful visual content.

SC 1.2.5 is the Level AA version of SC 1.2.3. The key difference:

  • 1.2.3 (Level A): Audio description OR text media alternative — either option satisfies
  • 1.2.5 (Level AA): Audio description is required — text alternative alone is no longer sufficient

Audio description benefits not just blind users but also people with cognitive disabilities who benefit from reinforced verbal descriptions of what they are watching.

Key Distinctions

Standard vs. Extended Audio Description
Standard AD inserts narration during existing pauses. Extended AD (SC 1.2.7, AAA) pauses the video to allow longer descriptions when there is insufficient pause time. For complex visual content, consider extended AD even if it is not strictly required.
Audio Already Describes the Visual — Exception
If the existing dialogue and sound fully convey all visual information, no separate audio description is required. The exception applies when the audio track is entirely self-describing — "as you can see" does not qualify.
Browser Support for kind="descriptions"
Most browsers render <track kind="descriptions"> as text, not spoken audio. Screen reader users read the cue text. An accessible media player (AblePlayer, MediaElement.js) is needed to speak the descriptions as audio to all users.

Examples

Example 1 of 4

Text Alternative Provided — But Level AA Requires Audio Description

A company website provides a text media alternative alongside their product video — this satisfies 1.2.3 (Level A). However, the site targets Level AA compliance. At Level AA (1.2.5), a text alternative alone is insufficient — an audio description track must also be provided.

Fails 1.2.5 Passes 1.2.3 only
✗ Level A only — Fails at Level AA
HTMLFAILS 1.2.5
<video controls>
  <source src="product.mp4">
  <!-- No <track kind="descriptions">
       Text alt alone passes 1.2.3 (A)
       but fails 1.2.5 (AA) -->
</video>

<a href="product-media-alt.html">
  Full media alternative
</a>

Problem: The text alternative approach meets Level A (1.2.3) but not Level AA (1.2.5). If your target conformance is AA — as required for most legal and procurement standards — you must add an audio description track.

✓ Audio Description Track Added — Satisfies 1.2.5
HTMLPASS 1.2.5
<!-- H96: audio description track
     satisfies 1.2.5 (AA) -->
<video controls>
  <source src="product.mp4">
  <track
    kind="descriptions"
    src="product-ad.vtt"
    srclang="en"
    label="Audio description"
  >
</video>

<!-- Text alt kept as supplement -->
<a href="product-media-alt.html">
  Full media alternative
</a>

Fix: Add <track kind="descriptions"> with a WebVTT file containing timed audio description cues. Keep the text alternative as a supplement — it helps users who prefer reading.

Example 2 of 4

Writing Effective Audio Description — WebVTT Examples

A training video shows a nurse performing a hand-washing procedure. The camera shows detailed hand movements that are not described by the voiceover. The audio description must fill these gaps precisely.

Technique: G78 Technique: H96
✗ Poor Audio Description — Too Brief
WebVTTINSUFFICIENT
WEBVTT

00:00:03.000 --> 00:00:05.000
Nurse washes hands.

00:00:14.000 --> 00:00:16.000
Uses soap.

<!-- Problems:
  - No detail about technique
  - Doesn't describe which steps
  - Missing timing (how long each step)
  - Fails to convey the clinical
    procedure being demonstrated -->

Problem: The description is technically present but conveys none of the specific information shown — the six-step WHO hand-washing technique, the circular motions, interlocking fingers, etc. It is equivalent to saying "slide shows data" for a chart.

✓ Effective Audio Description — Specific and Timed
WebVTTPASS
WEBVTT

00:00:02.000 --> 00:00:06.000
The nurse applies one pump of soap
to dry hands, then rubs palms together
in circular motions for 5 seconds.

00:00:12.000 --> 00:00:17.000
Interlacing fingers, they scrub between
each finger on both hands for 5 seconds.

00:00:22.000 --> 00:00:27.000
Cupping fingers, they scrub the backs
of both hands. The on-screen timer
reaches 20 seconds total.
🔊A blind trainee hears"The nurse applies one pump of soap to dry hands, then rubs palms together in circular motions for 5 seconds." — precise technique is conveyed

Good AD: Describe specific actions, body positions, visual text (like timers), and critical details. Use present tense. Be concise but complete. The description should allow someone to perform the task based solely on what they hear.

Example 3 of 4

On-screen Text — Titles, Labels, and Data Must Be Described

A corporate explainer video displays on-screen text: product names, statistics, and a callout quote. The voiceover does not read these out — it speaks generally about the topic while text appears on screen.

Rule: Every piece of on-screen text that is not also spoken in the audio track must appear in the audio description. This includes titles, statistics, captions, subtitles, and graphic text.

Example 4 of 4

Scene Changes and Setting Descriptions

A documentary film cuts between locations — a hospital ward, an outdoor market, and a government building. The audio track consists only of interview dialogue. Without scene descriptions, blind viewers have no sense of where each interview takes place, which is important context.

Technique: G78
✗ No Scene Setting
WebVTT — MissingFAIL
WEBVTT

<!-- The film cuts to a hospital ward.
     A nurse is interviewed.

     No AD cue describes the setting.
     The viewer only hears:
     "We see about 30 patients a day
     in our ward." They cannot tell
     where the speaker is or who
     is speaking. -->

Problem: Without a scene-setting description, blind viewers cannot understand the context — the hospital environment, the nurse's uniform, and the ward backdrop all convey important meaning that is lost.

✓ Scene and Speaker Described
WebVTTPASS
WEBVTT

00:04:02.000 --> 00:04:07.000
Cut to: a busy hospital ward. A nurse
in blue scrubs stands beside a row
of occupied beds. A caption reads
"Sister Amara Osei, Ward Manager."

00:04:08.000 --> 00:04:12.000
[Nurse speaks — AD pauses]

00:07:14.000 --> 00:07:18.000
Cut to: an outdoor street market.
Colourful stalls line both sides of
a narrow alley. A vendor gestures
toward shelves of produce.
🔊A blind viewer hears"Cut to: a busy hospital ward. A nurse in blue scrubs stands beside a row of occupied beds. A caption reads 'Sister Amara Osei, Ward Manager.'" — setting and speaker ID are clear

Good AD: Describe scene transitions, settings, key characters (including what they look like and on-screen name captions), and any meaningful visual activity during pauses in dialogue. Use concise, objective language.
